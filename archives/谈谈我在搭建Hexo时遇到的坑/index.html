谈谈我在搭建Hexo时遇到的坑

若想建站不踩坑，只能小心看脚下。

一号坑

这个坑基本是人人都会遇到，就是在配置文件“_config.yml”的“:”后面要一个空格！
当时我懵半天，哪不对呢……文件对了……主题也对了……卧槽哪错了啊！
然后注意到一个细节……配置文件的每个“:”都有一个空格……可我的主题配置没有……然后改了一下……
Git Bash→$ hexo d -g→username→password→Done
WTF什么鬼！尼玛一个空格危害世界啊！

二号坑

这个坑都是懒惹的祸，在更新文件时，懒得$ hexo clean直接$ hexo d -g然后，发现文件没更新=-=
琢磨半天，原来没clean啊！

三号坑

改主题文件，然后部署到Github上之后发现修改过的JS和CSS发现空白了！？
看了下本地的，里面明明有代码啊……
然后用了*二号坑*的clean之后……好惹=-=
